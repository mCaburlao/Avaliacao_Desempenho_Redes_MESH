## 🔧 TROUBLESHOOTING: E se...

### "Achei outro artigo sobre MESH que não estava na lista?"
→ Ótimo! Leia e veja se muda análise
→ Atualize MATRIZ manualmente
→ Recomendação deve permanecer a mesma (Linha 2) a menos que ache artigo explicitamente sobre segurança MESH

### "Meu advisor recomenda Linha diferente (ex: Linha 5)"
→ OK! Pegue o PLANO_Acao e adapte:
→ Semana 1-2: estudar artigo Lee (Linha 5) em vez de RFC 6550
→ Resto mantém estrutura
→ Metodologia experimental = mesma

### "Não consigo instalar Cooja"
→ Usar WSL2 (Windows Subsystem Linux 2) com Ubuntu
→ Ou usar cloud (ex: Google Colab) para simular depois
→ Não deixar isto bloquear; comece literatura enquanto instala

### "Tem 60 dias é muito pressionante"
→ não é "hard deadline"; é horizonte de planning
→ Se não terminar Dia 60? OK, vai para Dia 75, sem problema
→ Importante: fazer progresso steady, não speed

### "Quero contribuição mais radical (ex: nova arquitetura)"
→ Linha 2 = foundational (segurança é pré-requisito)
→ Após mestrado com Linha 2 → doutorado com inovação radical
→ Sequência recomendada: secure baseline first, then innovate
